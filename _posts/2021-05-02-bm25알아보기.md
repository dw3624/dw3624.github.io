이번 포스팅은 BM25에 관한 포스팅입니다. 이 모델은 tf-idf 모델의 tf 부분을 개선한 모델로 주로 검색 알고리즘에 쓰이는 모델입니다. tf-idf 모델에 대한 포스팅 링크는 [여기](https://dw3624.github.io/2021/04/11/tfidf%EC%95%8C%EC%95%84%EB%B3%B4%EA%B8%B0.html)입니다.

모델을 보기에 앞서 검색이란 무엇인지 정의를 합니다.
여기서 검색이란, 특정 검색단어(검색 쿼리)가 주어졌을때, 개별 문장에서 이와 일치하는 부분을 찾아내, 그 정도가 높은 순으로 배열하는 과정을 가리킵니다.

BM25는 위 검색 과정 중 일치율을 계산하는 부분에 주로 쓰입니다.



## 1. 기호 정의
수식 정의를 보기에 앞서, 기호를 정의하고 그 뜻을 알아보도록 합니다.

검색 쿼리 :
 - 검색하고자 하는 특정 단어의 집합입니다.
 - $$q = (q_1,q_2, ..., q_n)$$
 - "비둘기의 부리"의 경우 $$q_1 = 비둘기, q_2 = 의, q_3 = 부리$$


문장 데이터:
 - 검색쿼리를 검색하고자 하는 대상 문장 집합입니다.
 - $$D$$는 문장 전체 집합이며 개별 문장 $$d$$는 여기에 속합니다.
 - $$d \in D$$



## 2. 정의 & 의미
정의는 아래와 같습니다.
$$ score(q,d) = \sum_{i=1} idf(q_i) \times {( k_1 + 1) f(q_i,d )} \over {f(q_i,d) + k_1(1-b+b { { \# d} \over {avgdl} } ) }$$

각 기호의 의미는 아래와 같습니다.
 - $$idf$$ : 가중치합(등장이 잦을수록 낮은 점수 부여)
 - $$ \#d$$ : 문장 $$d$$의 길이, 문장 $$d$$에 들어있는 단어 개수
 - $$avgdl$$ : average document length, 문장별 평균 단어 개수
 - $$ \#d/avgdl$$ : 긴 문장에 대한 벌칙
 - $$ \sum $$: query 내 단어에 대해 계산되는 양의 총합
 - $$phi$$ : 일치율을 계산하기 위한 score
 - $$k_1$$: $$phi$$의 상한값($$phi$$는 $$k_1+1$$까지)
 - $${k_1} + { {1} / {k_1} }$$ : $$\#d = avgdl$$인 경우 $$f=1$$일때 $$phi =1$$
 - $$f( \dot )$$ : 문장 $$d$$ 속 검색 query $$q_i$$의 양
 - $$b$$ : 벌칙의 강도 조절
