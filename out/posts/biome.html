Biome

eslint와 prettier를 하나로 합친 듯한 툴 체인. 공식문서에서는 다음과 같이 소개하고 있다.

Biome is a fast formatter for JavaScript, TypeScript, JSX, and JSON that scores 97% compatibility with Prettier, saving CI and developer time.

Biome can even format malformed code as you write it in your favorite editor.

설치

프로젝트 디렉토리에 Biome 설치

아래 명령어로 root 디렉토리에 biome.json 생성한다. 일반적으로 package.json이 있는 디렉토리에 생성한다.

기본값은 다음과 같으며, linter와 formatter가 적용된다. formatter의 경우 기본적으로 적용이 돼 생략 가능하다.

package.json 수정

Biome의 linter 및 formatter는 CLI로 실행 가능하다. 다음과 같이 scripts를 추가한다. package.json에서 실행된 Biome은 해당 디렉토리에서 가장 가까운 biome.json를 찾아 적용한다.

  • format : --write 옵션으로 파일 및 디렉토리를 포매팅
  • lint : --apply 옵션으로 파일 및 디렉토리에 Lint 적용 및 수정 (eslint fix와 동일)
  • check : format, lint, import 정리 실행

적용

vscode에서 Biome을 사용하려면 defaultFormatter 설정을 바꿔야 한다.

  1. vscode 확장기능 설치
  2. settings.json 수정
    • 설정 -> 설정(JSON) 열기

editor.codeActionsOnSave에 quickfix.biome을 설정해 저장 시 자동수정이 적용된다. source.organizeImports.biome으로 import 정렬이 적용된다.

Biome 설정 수정

설정 파일을 편집해 적용할 formatter와 linter를 선택할 수 있다. override로 특정 파일에만 Biome을 적용하도록 할 수 있다. Next.js의 경우 export default가 필요한 파일이 있기 때문에 해당 파일에 대해서 noDefaultExport를 off로 설정한다.

Linter

linter 기능을 제공한다. Biome Linter

Biome이 권장하는 기본값이 설정돼 있다. error, warn, off를 설정해 규칙 적용 방식을 변경할 수 있다.

다음과 같이 주석을 추가해 lint을 무시할 수 있다.

  • biome-ignore: 무효 주석 시작
  • lint: linter 무효화
  • /suspicious/noDebugger: 무효화할 규칙의 그룹과 이름 (선택)
  • <explantation>: 규칙을 무효화하는 이유
